## Propósito

Esta residencia es mi proceso público de formación para convertirme en un **Founding AI Engineer**: una persona capaz de tomar una idea, convertirla en producto, desplegarla, venderla, mejorarla y operarla con criterio técnico y visión de negocio.

No estoy siguiendo un curso tradicional.

Estoy construyendo una residencia práctica basada en proyectos reales, documentación, iteración y exposición pública del aprendizaje.

La meta no es solamente aprender a usar Claude o cualquier otra herramienta de inteligencia artificial.

La meta es desarrollar la capacidad de construir.

> Las herramientas cambian.
> Las capacidades permanecen.

## Lo logrado en E0

En E0 completé el ciclo fundamental de construcción:

* Crear una carpeta nueva desde cero.
* Inicializar un repositorio con Git.
* Crear archivos base como `README.md` e `index.html`.
* Revisar el estado del proyecto con `git status`.
* Preparar cambios con `git add`.
* Crear un commit con un mensaje claro.
* Crear un repositorio público en GitHub usando `gh repo create`.
* Conectar el repositorio local con GitHub.
* Subir cambios con `--push`.
* Desplegar un sitio estático en Vercel.
* Verificar que el sitio estuviera vivo en una URL pública.
* Limpiar recursos desechables después de la prueba.

El aprendizaje más importante de E0 no fue memorizar comandos.

Fue entender el bucle completo:

> **carpeta → archivo → Git → commit → GitHub → deploy → URL pública**

Ese ciclo es la unidad mínima de construcción visible.
